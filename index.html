向默默改变自己的人致敬

一年多以前，新公司刚入职第一天。按照wiki新员工手册搭建开发环
境，其他一切都很顺利，唯独拉项目代码时候，git怎么都登不上去，
检查用户名密码翻来覆去好几遍，仍然无果。这时候，跟我隔着一排坐
的小伙凑过来，跟我说了个新的密码让我试一试，将信将疑试了一下，
果然成功。当时，对他印象也不是很深刻，只当是个热心肠的同事。

当时业务比较忙，公司的客服电话一天会多次响起，碰巧客服人员刚好不在的时候，刺耳的电话铃声就在办公室回荡，研发们也都事不关己高高挂起，沉浸在自己的事情中，对电话铃声置若罔闻。那个时候只要他在办公室，就会第一时间接起电话，当然一开始作为客服的他，并不是很专业，很多问题要么不知道要么回答的磕磕绊绊。我相信当时肯定很多人跟我抱着一样的想法，一方面嫌弃他多管什么闲事，另一方面暗自嘲笑他并不流利的表达。

让我留意到他的除了充当客服人员这一角色，还有就是每天的站立会或者时不时的头脑风暴各种会，大多数参会人都是抱着应付的心态，各顾各的在那玩手机或者三言两语说说自己的想法应付了事。只有他虽然说得不好，虽然略显冗余，但是每次都积极主动在众人不厌其烦的表情中，一次次认真细致的表达自己的想法。

时间久了，效果就不一样了。不知从什么时候起，大家都对他这个新的“客服人员”习以为常，对他在站立会上的各种长篇大论也不再是不厌其烦转而是饶有兴趣的听着。而他则表达越来越流利，应付各种突发情况越发的游刃有余，开会的时候总能说出很多闪光点，不光有想法还有实现方案还有各种细节…..

相反，这段时间过去了，我们还是我们，还是一如既往的旁观他不停的改变，而我们呢，或许只是原地踏步，或许已经有了明显的后退，唯一不变的是还是用自己的一点小聪明一点小耍滑，自以为是的看待别人的不断进步。

一年多过去了，我这个“原地踏步”的人，这么久了，才想着贴上：

阅读-NodeJs.相关

Node.js是什么？

Node.js，或者 Node，是一个可以让 JavaScript 运行在服务器端的平台。它可以让 JavaScript 脱离浏览器的束缚运行在一般的服务器环境下，就像运行 Python、Perl、PHP、Ruby 程序一样。你可以用 Node.js 轻松地进行服务器端应用开发，Python、Perl、PHP、Ruby 能 做的事情 Node.js 几乎都能做，而且可以做得更好。

Node.js 是一个为实时Web(Real-time Web)应用开发而诞生的平台，它从诞生之初就充分 考虑了在实时响应、超大规模数据要求下架构的可扩展性。这使得它摒弃了传统平台依靠多线 程来实现高并发的设计思路，而采用了单线程、异步式I/O、事件驱动式的程序设计模型。这些 特性不仅带来了巨大的性能提升，还减少了多线程程序设计的复杂性，进而提高了开发效率。
